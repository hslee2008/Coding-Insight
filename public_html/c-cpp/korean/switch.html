조건문이 복잡해진다!!!

조건문이 매우 복잡해질 때가 있습니다.

예를 들면, char를 입력 받아 A이면 1, B이면 2, C이면 3그리고 F이면 6까지 출력하는 프로그램을 만들어 보겠습니다.

#include <stdio.h>

int main()
{
  char a;
  scanf("%d", &a);

  if(a == 'A') {
    printf("1");
  } else if(a == 'B') {
    printf("2");
  } else if(a == 'C') {
    printf("3");
  }else if(a == 'D') {
    printf("4");
  } else if(a == 'E') {
    printf("5");
  } else if(a == 'F') {
    printf("6");
  }

  return 0;
} 흠... 조금 이해하기 어렵습니다. 이 상황에서 switch문을 사용하면 어떨까요?



switch 문

1. switch() {}를 선언한 후 괄호() 안에 비교하고 싶은 변수를 넣습니다.

(지금부터 중괄호{} 안에 코드를 넣습니다.)

2. case "A":라고 친 후 새로운 줄에 만일 a의 값이 A라면 샐행할 코드를 넣습니다.

(참고로 case는 {} 중괄호를 사용하지 않고 ':'를 사용합니다)

4. 그리고 항상 a의 값이 A라면 실행할 코드 다음에 "break"를 넣어야 합니다.

#include <stdio.h>

int main()
{
  char a;
  scanf("%c", a);

  switch(a) {
    case 'A':
      printf("1");
      reak;
    case 'B':
      printf("2");
      break;
  }

  return 0;
}
